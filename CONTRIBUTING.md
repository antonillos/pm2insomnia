# Contributing

## Before opening a pull request

```bash
pip install -e ".[dev]"
make check
```

## Pull requests

- Keep changes focused and small when possible.
- Add or update tests when behavior changes.
- Update `README.md` when user-facing behavior or CLI usage changes.
- Prefer generic examples and fixtures over private or local-specific data.

## Reporting issues

- Use GitHub Issues for bugs, feature requests, and usability feedback.
- Do not use public issues for security reports. Follow `SECURITY.md` instead.
